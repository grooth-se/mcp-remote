Report Approval Workflow - Help

How a test report goes from raw test data to a published, signed PDF certificate.

Workflow Overview
Draft Pending Review Published
  1. Create the certificate and run the tests linked to it (Certificates → New).
  2. Generate the Word report from the certificate page, or upload your own .docx. The report is stored on the server with status Draft.
  3. Edit if needed: download the Word file, edit it locally, and upload it back. Each upload replaces the working copy on the server.
  4. Submit for approval. The report is locked (Pending Review) and appears in the approvers' Reports queue.
  5. An approver reviews the certificate data, test results, and the Word document, then either:
    • Approve & Sign - the system stamps the approval into the report, converts it to PDF, appends any PDF attachment, and signs it with the company X.509 certificate. Status becomes Published.
    • Reject - with mandatory comments. Status becomes Rejected and the report goes back to the engineer.
  6. Download the published PDF from the certificate page or the report register. Published PDFs are immutable; a SHA-256 hash is stored for integrity verification.
Who Can Do What
Role User ID Permissions in the approval workflow
Operator DUR-OPR-xxx Run tests and view data. Cannot create or submit reports.
Test Engineer DUR-ENG-xxx Generate/upload/edit reports and submit them for approval.
Approver DUR-APR-xxx Everything an engineer can, plus review, approve & sign, reject, and replace published PDFs.
Administrator DUR-ADM-xxx All of the above, plus user management and certificate deletion.
Four-eyes principle: the person who submitted a report cannot approve it, even with the approver role. A second approver must review and sign.
Step by Step: Test Engineer
  1. Open the certificate (Certificates → click the certificate number).
  2. In the Report Approval card, click Generate Word Report to create the report from test data, or upload an existing .docx instead.
  3. Optional: click Download Word for Editing, adjust the document in Word, then use Upload Edited to save it back.
  4. Click Submit for Approval. The report is now locked - you cannot upload new versions while it is pending.
  5. If the report is Rejected, the rejection reason is shown on the certificate page. Fix the Word document, upload it, and re-submit.
Step by Step: Approver
  1. Open Reports → Pending Review (a red badge in the navbar shows how many reports are waiting).
  2. Click Review on a report. Check the certificate data, the test results, and download the Word document to verify its content.
  3. Choose one:
    • Approve & Sign - confirm on the signing page. The system creates the signed PDF automatically.
    • Manual PDF upload (fallback) - if automatic conversion/signing is not available, upload a PDF you signed with external software. The uploaded file must be complete, including any attachment pages.
    • Reject - enter a comment explaining what must be corrected.
Signing requirements: automatic signing needs LibreOffice (Word→PDF conversion), the endesive library, and the company .p12 certificate in the certs/ folder. The signing page shows which of these are available. Without a certificate, the PDF is published unsigned (a warning is shown).
Frequently Asked Questions
A published report contains an error. How do I fix it?
Published reports are immutable. Open the certificate and click Create Revision - this creates a new certificate revision (e.g. DUR-2026-1001 Rev.2) with its own report workflow that supersedes the original. Approvers can also Replace signed PDF on the certificate page if only the PDF file itself must be swapped (the replaced file is archived).
Why can't I approve a report I submitted?
ISO 17025 requires independent review. Ask another approver to review it.
Why is the Submit button missing?
Either no Word report has been generated/uploaded yet, the report is already pending or published, or your role does not allow submitting (operators cannot submit reports).
Where are the files stored?
Draft Word documents: reports/drafts/. Published PDFs: reports/signed/<year>/. Replaced PDFs are archived in reports/archive/<year>/.
What does the PDF attachment do?
If a test record has a PDF attachment (e.g. raw data or external results), its pages are automatically appended to the report PDF before signing, so the published document is complete and the signature stays valid.
Is everything logged?
Yes. Every workflow action (start, upload, submit, approve, reject, PDF replacement) is written to the audit trail with user, timestamp, and IP address.
